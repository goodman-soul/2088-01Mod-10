# 三进程乒乓游戏（无名管道 + 信号同步）

本程序创建 3 个进程：父进程、子进程 1、子进程 2。
使用 **一个无名管道** 在三个进程之间传递整数，并通过信号做严格轮转同步。

- 父进程先发送 `0` 给子进程 1
- 子进程 1 加 1 后回传给父进程
- 父进程加 1 后发送给子进程 2
- 子进程 2 加 1 后回传给父进程
- 循环往复

打印规则：
- 每个进程打印的都是"自己加 1 后"的值
- 使用 `fprintf` 打印
- 每次打印后都 `fflush(stdout)`

## 多局比赛模式

程序支持多局比赛，每局会重新创建管道与子进程，状态完全隔离。
启动后依次输入以下参数：

| 参数 | 说明 |
|------|------|
| 局数 | 总共打多少局（正整数） |
| 最大传球次数 | 每局累计传球达到该次数即判胜（正整数） |
| 超时毫秒 | 每局独立倒计时，超时判负（正整数） |
| 随机失败概率 | 每次传球以该概率触发随机失败判负（0–100） |

## 每局可能的结果

- **胜利（达到最大传球次数）**：累计传球次数达到上限
- **失败（超时）**：超过设定的毫秒数
- **失败（随机失败）**：任意一方触发随机失败
- **中断**：收到外部信号等异常情况

## 比赛总结

所有局结束后输出：
- 总局数 / 已玩局数
- 胜局数、负局数、胜率
- 平均传球次数
- 失败类型分布（超时、随机失败、中断）

## 本地运行

```bash
make
./pingpong
```

示例交互：

```
请输入局数：5
请输入最大传球次数：100
请输入超时毫秒：1000
请输入随机失败概率(0-100%)：10
```

## Docker Compose 一键运行

```bash
docker compose build
docker compose run --rm pingpong
```

停止并清理：

```bash
docker compose down
```
